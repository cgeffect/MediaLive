# encoder-pusher 代码速读文档

本文档用于快速理解 `src/encoder-pusher` 的代码结构与执行链路，目标是做到：

- 10 分钟内定位主流程
- 30 分钟内能独立排查常见问题
- 面试时能清晰讲出模块设计

## 1. 模块目标

`encoder-pusher` 负责将输入媒体转换为实时预览流，并推送到流媒体服务（ZLMediaKit）。

当前支持两种路径：

- 文件输入路径：`decode -> frame render -> encode -> rtmp push`
- 无输入兜底路径：`ffmpeg testsrc/sine -> rtmp push`

并通过 `streamId` 支持多流并发任务。

## 3. 一条流从启动到推流的调用链

### 3.1 脚本层

- `start_stream.sh <streamId> ...`
  - 解析 `streamId`
  - 准备二进制（优先顶层 CMake 产物）
  - 启动 `encoder_pusher`，将 pid/log 写入 `state/<streamId>.pid|log`

### 3.2 应用入口层

- `src/app/main.cpp`
  1. `ParseArgs(argc, argv)` 得到 `TaskConfig`
  2. `TaskManager.StartTask(cfg)` 启动任务线程
  3. 主线程等待退出信号，最终 `TaskManager.StopAll()`

### 3.3 任务调度层

- `src/core/task_manager.cpp`
  - `StartTask`：按 `streamId` 创建任务
  - `RunTask`：每个任务线程内创建 `FrameRenderPusher`
  - `ListTasks`：打印当前所有流
  - `StopAll`：停止并 join 全部线程

### 3.4 Pipeline 编排层

- `src/pipeline/frame_render_pusher.cpp`
  - `RunFrameRenderOnce(running)`：
    1. `InitInput(...)`
    2. `InitOutput(...)`
    3. `av_read_frame` 循环读包
    4. 视频包 -> `ProcessVideoDecode(...)`
    5. 音频包 -> `ProcessAudioCopy(...)`
    6. EOF -> `FlushVideo(...)`
  - `RunCliFallback(...)`：无输入时执行 testsrc/sine 兜底

## 4. 各文件职责（速记版）

- `core/task_config.cpp`
  - 命令行参数解析
  - 拼接 `rtmp://host:port/app/streamId`

- `pipeline/input_initializer.cpp`
  - 打开输入文件
  - 查找视频/音频流
  - 初始化视频解码器

- `pipeline/output_initializer.cpp`
  - 创建 FLV/RTMP 输出
  - 初始化 H.264 编码器
  - 创建输出流并写 header

- `pipeline/video_processor.cpp`
  - 解码视频包为帧
  - 必要时做像素格式转换（sws）
  - 调用 `frame_renderer` 做帧处理
  - 编码后写出

- `pipeline/audio_processor.cpp`
  - 当前为 AAC 音频透传写出

- `pipeline/frame_renderer.cpp`
  - 逐帧视觉效果（当前为亮带示例）
  - 这里是你替换业务渲染逻辑的核心入口

- `pipeline/ffmpeg_utils.cpp`
  - FFmpeg 错误码转字符串
  - 根据时间戳控制发送节奏（实时感）

- `pipeline/pipeline_context.cpp`
  - 统一回收 FFmpeg 资源（RAII）

## 5. 关键参数与行为

- `--stream-id`
  - 决定推流名与拉流 URL
- `--input`
  - 指定输入文件时走帧级 pipeline
  - 不指定时走 testsrc/sine 兜底
- `--host --port --app`
  - 决定 RTMP 目标地址

对应拉流地址：

- `http://<zlm_host>:8081/live/<streamId>.live.flv`

## 6. 你最常改的点

1. 替换渲染效果
   - 文件：`pipeline/frame_renderer.cpp`
2. 新增视频滤镜链
   - 文件：`pipeline/video_processor.cpp`
3. 升级音频处理（解码/混音/编码）
   - 文件：`pipeline/audio_processor.cpp`
4. 扩展任务策略（重试/超时/限流）
   - 文件：`core/task_manager.cpp`

## 7. 常见问题排查

### Q1: 有画面没声音

- 先看 Web 播放器是否 `muted`
- 再看输入文件是否带 AAC 音轨（当前音频处理按 AAC 透传）

### Q2: 拉流 404 或无流

- 确认 `streamId` 是否一致
- 看 `state/<streamId>.log` 是否启动成功
- 看 `src/media-server/zlm.log` 是否收到发布流

### Q3: 启动脚本报找不到二进制

- 先执行顶层构建：
  - `cmake -S . -B build`
  - `cmake --build build -j "$(sysctl -n hw.logicalcpu)"`

## 8. 面试可直接讲的 60 秒版本

“我把推流模块拆成了 app/core/pipeline 三层：入口层只负责参数和生命周期；core 层做 streamId 任务调度；pipeline 层负责媒体主链路。视频链路是解码、帧级处理、编码再推流，音频当前做 AAC 透传。这样设计的好处是渲染逻辑和任务管理解耦，后续可以很容易扩展多轨合成、滤镜和音频混音。”
